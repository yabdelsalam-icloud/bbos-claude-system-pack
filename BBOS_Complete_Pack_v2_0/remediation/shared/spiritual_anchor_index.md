# BBOS Spiritual Anchor Index
@@DOC:BBOS_SPIRITUAL_ANCHOR_INDEX@@

**Purpose:** Operators cannot know in advance whether an ANNOTATE command will succeed or return NO_SHIP without knowing which sections of each stage contain a confirmed spiritual anchor. This index documents every valid annotation placement point across all stages. If a section is not listed here, it does not have a spiritual anchor — ANNOTATE invoked against it will return NO_SHIP.

---

## How to use this index

Before invoking `ANNOTATE`, check this index for the active stage. Confirm that the section or asset you want annotated is listed. Then invoke:

```
ANNOTATE [Stage XX] [Section or Asset name as listed below]
```

If you invoke ANNOTATE for a section not listed, the governor returns NO_SHIP with: "No spiritual anchor exists for [section]. Annotation cannot be placed."

## Annotation constraints (applies to all stages)

- Maximum 2 sentences per annotation.
- One specific operational implication per annotation.
- Never repeat the same divine attribute twice in the same output.
- Never use spiritual language to reinforce urgency, scarcity, or closing pressure.
- Annotations are operator-facing only — they are never placed in client-facing copy without the operator's deliberate choice.
- If a requested annotation point is not in this index, return NO_SHIP rather than inventing a placement.
